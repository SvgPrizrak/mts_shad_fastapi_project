## Структура проекта

Для удобства и соблюдения принципов чистой архитектуры проект разделен на следующие пакеты:

- `configurations` — слой для хранения конфигураций, констант, параметров и настроек проекта;

- `models` — слой для хранения моделей (ORM или Data Classes);

- `routers` — слой для настроек урлов для различных эндпоинтов c учетом авторизации;

- `schemas` — слой содержащий схемы pydantic, отвечает за сериализацию и валидацию;

- `services` — слой содержащий бизнес-логику и работу с ORM.

## Запуск

Запуск осуществлялся на операционной системе `Windows 11` с предустановленными `Docker Desktop`, `Virtual Studio Code` (или иной IDE), `DBeaver` (или иным менджером баз данных) и виртуальным окружением `Python 3.13.9`. Перед проверкой работоспособности требуется:
1. Склонировать репозиторий через `git clone`;
2. Создать в корневой папке `src` файл `.env` по примеру из `.env_example`;
3. Создать виртуальное окружение Python: `python -m venv .venv`;
4. Активировать окружение (вариантов много, я запускал через `PowerShell`: `.\\.venv\Scripts\activate`, есть ещё вариант запуска через `WSL` и `bash`);
5. Установить зависимости внутри (эти и последующие команды запускать из корневой папки `shad_fastapi_project_2026`) виртуального окружения: `pip install -r requirements.txt`;
6. После формирования контейнера запустить команду `uvicorn src.main:app --reload`;
7. В корневой папке запустить команду `docker compose -d` (не забыть предварительно запустить сам `Docker`, если контейнер запускается в первый раз, то произойдет сборка образа);
8. Создать 2 базы данных в менеджере баз данных (основную - `fastapi_project_db` и тестовую - `fastapi_project_test_db`) с параметрами подключения, указанными в `.env` файле;
9. Проверить работоспособность репозитория через файл `api_tests.http` и/или через тесты `pytest src/tests -v`.
10. PROFIT
